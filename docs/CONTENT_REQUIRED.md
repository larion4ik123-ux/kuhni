# CONTENT_REQUIRED — недостающие данные клиента

Все неизвестные значения хранятся в конфигурации (админпанель / `.env`) и
отмечены здесь. Заполнить до публикации.

## Критично (без этого нельзя публиковать)

- [ ] **Фотография владельца** (Артём Ермаков) — вертикальная, качественная,
      нейтральный фон. Сейчас плейсхолдер.
- [ ] **Логотип** «Мебельный салон Интерьер» — вектор/PNG с прозрачным фоном.
- [ ] **Фото выполненных кухонь** (минимум 6-12) с метаданными для каждого:
      планировка (прямая/угловая/П-образная/с островом), стиль
      (современный/неоклассика/классика), основной цвет, краткое описание,
      alt-текст. Это основа галереи и первого экрана.
- [ ] **Telegram-бот:** username и token (через @BotFather). URL бота для
      deep links сайта.
- [ ] **API-ключ polza.ai** (или иного провайдера) и подтверждение модели
      для img2img (по умолчанию `bytedance/seedream-4.5`).
- [ ] **Контакты:** телефон, регион работы, часы связи, физический адрес.

## Желательно

- [ ] **Образцы материалов:** фасады (Interno, AGT, ...), цвета, ручки,
      столешницы — фотографии/рендеры образцов для блока материалов и как
      референсы для AI.
- [ ] **Отзывы:** реальные отзывы с Яндекс Карт (имя, текст, оценка, дата,
      ссылка на оригинал) — для ручного внесения; либо код официального
      виджета Яндекс Карт для вставки iframe.
- [ ] **Ссылка на Яндекс Карты** (профиль компании) для кнопки «Посмотреть
      все отзывы на Яндекс Картах».
- [ ] **Менеджеры:** chat_id в Telegram для получения уведомлений о заявках.

## Факультативно

- [ ] Подтверждённый стаж/опыт (если хочется указать «N лет на рынке») —
      **указывать только подтверждённое**, иначе блок скрыт.
- [ ] Число выполненных заказов — только подтверждённое, иначе скрыто.
- [ ] Портфолио «до/после» (если есть) — для блока генератора/работ.
- [ ] Будущий **MAX**-аккаунт бота (для этапа 2).

## Политика

- Фото клиента → `assets/raw/<категория>/`, затем `scripts/process_images.py`.
- Любые новые данные (отзывы, тексты, контакты) → через админпанель в БД,
  не в код.
- AI-визуализации всегда подписаны как предварительные; никогда не помечать
  `is_real_work=true` для сгенерированных изображений.
