# FUNNEL — настраиваемая воронка подбора кухни

Воронка **не зашита в код**. Вопросы и варианты хранятся в БД
(`funnel_questions`, `funnel_options`) и редактируются в админпанели.
Бизнес-логика в `backend/app/services/FunnelService` мессенджер-независима;
мессенджер только рендерит текущий вопрос и принимает ответ.

## Структура вопроса

| Поле | Назначение |
|---|---|
| `key` | уникальный идентификатор (`form`, `style`, `color`, ...) |
| `title` | текст вопроса |
| `description` | пояснение под вопросом |
| `type` | `single_choice \| multiple_choice \| text \| number \| range \| photo \| contact \| confirmation` |
| `media_file_id` | изображение к вопросу (необязательно) |
| `order` | порядок |
| `required` | обязательный |
| `active` | включён |
| `allow_skip` | можно пропустить |
| `generation_key` | ключ для подстановки в AI-промпт (например `style`) |
| `condition` | JSON: когда показывать (см. «Условия») |
| `next_question_rule` | JSON: какой вопрос следующий (по умолчанию — следующий по order) |

## Вариант ответа

| Поле | Назначение |
|---|---|
| `title` | видимый текст («Современный») |
| `description` | пояснение |
| `media_file_id` | изображение варианта |
| `internal_code` | код для логики (`modern`, `corner`, ...) |
| `generation_hint` | **текст для AI-промпта** (не только название) |
| `active`, `order` | видимость и порядок |

Пример: `title="Современный"`, `generation_hint="Современная кухня с чистыми
геометрическими линиями, гладкими фасадами, лаконичной встроенной техникой и
минимальным декором"`. AI получает `generation_hint`.

## Начальный сценарий (seed)

1. Приветствие (`text`, не сохраняется как выбор)
2. Форма кухни (`single_choice`: прямая / угловая / П-образная / с островом)
3. Стиль (`single_choice`: современный / неоклассика / классика)
4. Цвет фасадов (`single_choice`: серый / бежевый / зелёный / белый)
5. Фасад (`single_choice`: Interno / AGT / другие — из админки)
6. Ручки / система открывания (`single_choice`: скоба / чёрная квадратная /
   кнопка / профиль Gola)
7. Столешница (`single_choice`, **active=false** по умолчанию — включается в админке)
8. Фурнитура (`single_choice`, **active=false**)
9. Размер / площадь (`text` / `number`)
10. Желаемый срок (`text`)
11. Бюджет (`range`/`text`, **active=false**)
12. Дополнительные пожелания (`text`, `allow_skip=true`)
13. Тип исходного помещения (`single_choice`: пустое / старая кухня)
14. Загрузка фотографии (`photo`, `required=true`)
15. Просмотр итогового выбора (`confirmation`) + изменение ответов
16. Запрос контакта (`contact`) ← **до этого генерация НЕ запускается**
17. Создание заявки + уведомление администратора/менеджеров
18. Постановка job генерации, если настроен провайдер и есть контакт
19. Менеджер получает подбор и связывается с клиентом по следующему шагу

Поля `active=false` (столешница, фурнитура, бюджет) — отключены по умолчанию,
включаются в админпанели без правки кода.

## Условия (`condition`)

JSON-объект, определяющий, когда показывать вопрос. Пример:
```json
{"question_key": "form", "option_code_in": ["corner", "u_shape"]}
```
— вопрос показывается только если на `form` выбран угловой или П-образный.
Пустое условие → показывается всегда. Логика в `FunnelService.should_ask()`.

## Навигация (`next_question_rule`)

По умолчанию — следующий активный вопрос по `order`. Переопределение:
```json
{"on_option": "classic", "next_question_key": "facade_classic"}
```
или `{"next_question_key": "color"}` (безусловный переход).

## Источники (deep links)

- `start=website_main` — общий вход
- `start=website_generator` — блок генератора
- `start=website_gallery` — из галереи («похожую кухню»)
- `start=organic` — органический заход

Источник сохраняется в `funnel_sessions.source` и `leads.source`.

## Критичный инвариант генерации

`GenerationService.create_job()` проверяет наличие `user_contacts` у пользователя.
Без контакта — генерация не ставится в очередь; пользователю показывается:
«Остался последний шаг: отправьте номер. Бот передаст заявку администратору,
а менеджер свяжется с вами по следующему шагу».

## Формирование описания и промпта

На основе ответов формируется:
1. **Описание выбора** (для менеджера и заявки): «Клиент выбрал угловую кухню
   в современном стиле, с серыми фасадами AGT, чёрными квадратными ручками,
   светлой столешницей и фурнитурой Blum.» — человекочитаемый текст.
2. **Generation prompt**: мастер-промпт (из настроек) + `generation_hint`
   каждого выбранного варианта + параметры (размер, срок, пожелания, тип
   помещения) + инструкция сохранить помещение (стены/окна/двери/пол/потолок/
   перспективу/освещение), убрать старую кухню, разместить новую реалистично,
   без людей, не менять архитектуру.

Подпись результата: «Визуализация является предварительной и служит для
подбора идеи». Не обещать точный проект/чертёж.
